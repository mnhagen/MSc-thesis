\chapter{\label{cha:method}Experimental Setup}

This chapter describes the datasets, evaluation metrics, and training
procedures used throughout the thesis. It also clarifies how models
with different output-resolution behavior are compared in the
discretization transfer experiments.

\section{Datasets}

The experiments in this thesis use both uniform-grid and non-uniform
datasets. For each dataset, the goal of this section is to describe the
underlying physical problem, the mapping learned by the model, the data
source, and the aspects of the dataset that are relevant for
interpreting the results. Dataset-specific implementation details are
included only when they affect the conclusions drawn later in the
thesis. In particular, the full generation procedure for the
self-generated Navier-Stokes dataset is deferred to Appendix~X.

\subsection{Uniform-grid datasets}

The discretization invariance experiments on regular grids use two
datasets: a self-generated Navier-Stokes dataset and a Darcy flow
dataset obtained from prior work.

\subsubsection{Navier-Stokes}\label{navier-stokes}

\begin{figure}
    \centering
    \includegraphics[width=0.8\textwidth]{figures/navier_input_output.png}
    \caption{\label{fig:navier} Initial and final vorticity fields for a sample from the Navier-Stokes dataset.}
\end{figure}

For the regular-grid experiments, we use a self-generated two-dimensional Navier-Stokes dataset on the unit square. The Navier-Stokes equations are among the fundamental equations of fluid dynamics and arise in many areas of physics and engineering, including atmospheric and oceanic flows, aerodynamics, and industrial transport processes. The dataset is formulated and solved in vorticity form, with governing equation:


\begin{equation}
    \partial_t \omega + u \cdot \nabla \omega = \nu \Delta \omega + f,
\end{equation}

where $\omega$ denotes the vorticity field, $u$ the incompressible velocity field, $\nu$ the viscosity, and $f$ an external forcing term. The learning task considered in this thesis is not a pressure or velocity prediction problem, but an operator mapping from an initial vorticity field to a later vorticity field. The input to the model is the initial vorticity field $\omega(\cdot,0)$, and the output is the vorticity field at a later time $\omega(\cdot,T)$. An example input-output pair is shown in Figure~\ref{fig:navier}.

The dataset consists of 1000 samples generated at a resolution of $256 \times 256$. Between samples, the viscosity and forcing remain fixed, while the initial vorticity field is varied.

Further details on the data-generation procedure are provided in Appendix~\ref{navier-details}.

\subsubsection{Darcy flow}

Also used for the uniform grid experiments is the Darcy flow dataset introduced by Li et al.\cite{li2021fno}. 
In this benchmark, the task is to learn the solution operator of a steady-state two-dimensional Darcy flow problem
on the unit square, governed by the following PDE:

\begin{figure}
    \centering
    \includegraphics[width=0.8\textwidth]{figures/darcy_input_output.png}
    \caption{\label{fig:darcy} Example of an input diffusion coefficient field \(a(x,y)\) and the resulting pressure
    field \(u(x,y)\) with Dirichlet boundary conditions.}
\end{figure}

\begin{equation}
    -\nabla \cdot (a(x)\nabla u(x)) = f(x), \qquad x \in (0,1)^2,
\end{equation}
\begin{equation}
    u(x) = 0, \qquad x \in \partial(0,1)^2,
\end{equation}

 The input is a spatially varying diffusion coefficient field $a(x)$, and the output is the
 corresponding solution field $u(x)$ under a prescribed forcing term and Dirichlet boundary conditions. An example
 input-output mapping can be seen in figure \ref{fig:darcy}. Between samples, $f(x)$ is kept constant while the $a(x)$ is varied. This PDE has many applications within physics and engineering, including drainage and irrigation in agricultural soils and fluid migration around nuclear waste repositories \cite{darcyapplications}. For this 
  thesis, the Darcy flow dataset serves as a regular-grid benchmark for evaluating the performance of FNO and CNO 
  across discretizations. This dataset was selected because it provides a comparatively smooth benchmark, 
  complementing the more irregular and turbulent behavior of the Navier-Stokes dataset discussed in Section~\ref{navier-stokes}. Using both datasets makes it possible to evaluate model performance under qualitatively different solution regimes. The dataset contains 10000 samples at a resolution of 128$\times$128.



\subsection{Non-uniform datasets}

The experiments on irregular domains use three datasets: the elasticity
benchmark used in the original geo-FNO work, an incompressible Euler
dataset modeling flow past a cylinder, and a nuclear engineering
dataset consisting of temperature profiles for a single HTGR column.

\subsubsection{Elasticity}

The elasticity dataset is included because it is one of the canonical
benchmarks used to evaluate geo-FNO on non-uniform meshes. It provides
a direct connection to the original geo-FNO setting and therefore helps
assess whether the implementation used in this thesis reproduces the
expected behavior on an established test case.

In the dataset description, it is enough to explain the physical problem
at a high level, describe the input and output quantities, and note that
the data is defined on an irregular geometry rather than a uniform
Cartesian grid. The main text should also cite the original source and
report the experimental resolution or mesh representation at the level
needed to interpret the results.

\subsubsection{Incompressible Euler flow past a cylinder}

A second non-uniform benchmark is an incompressible Euler dataset
describing fluid flow past a cylinder. This dataset was used in prior
work on operator learning for PDEs and is included here because it
provides a fluid-dynamics test case on a geometry that is not naturally
aligned with a regular grid.

For the purposes of this thesis, the most important information is what
the model receives as input, what field it predicts, and why the domain
geometry makes this dataset relevant for geo-FNO. As with the elasticity
dataset, the source should be cited and the description should include
the mesh or sampling representation used in the experiments. Fine-grained
details of the original data-generation process are not needed unless
they become important for later analysis.

\subsubsection{HTGR column temperature dataset}

The final non-uniform dataset consists of temperature profiles for a
single column of a high temperature gas-cooled reactor (HTGR) and was
generated by a colleague. This dataset is of particular interest because
it connects the thesis to the motivating nuclear engineering
application. Compared with the public benchmark datasets, it is also a
more domain-specific test of whether geo-FNO can learn useful operators
for reactor-related data.

In the main text, this dataset should be introduced in terms of its
physical meaning, the variables that define the input and output, and
the way spatial locations are represented. Since the dataset is not a
standard public benchmark, it is also worth briefly explaining how it
was obtained and why it is scientifically relevant. If there are strong
structural regularities or simplifications in the data that later help
explain the results, those should be noted here as well.

\subsection{Scope note}

In addition to the datasets discussed above, a non-uniformly sampled
Navier-Stokes dataset was used during troubleshooting. Since it was not
part of the final evaluation, it is not described as a primary dataset
in this thesis.
